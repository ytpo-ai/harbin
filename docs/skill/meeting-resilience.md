# Meeting Resilience

## 1. 触发条件

- 当前任务为会议类任务（`meetingLikeTask=true`）。
- 命中以下任一异常：
  - 空回复（空字符串、仅符号、无意义占位）；
  - 生成异常（超时、临时不可用、网络抖动、限流）；
  - 上游长耗时且当前轮无法形成可交付文本。

## 2. 异常分级与处理

### 2.1 空回复

1. 立即执行一次重试。
2. 重试仍为空时，返回最小可用回执。

最小可用回执结构：

- 已分配：说明当前分配动作或待分配对象。
- 已通知：说明已通知的对象或通知状态。
- 下一检查点：给出明确时间或条件（例如“1 分钟后补充结果”）。

### 2.2 生成异常

1. 若属于暂时性异常，执行一次重试，不重复追问用户。
2. 重试失败则直接返回可执行回执，并标注失败原因。

### 2.3 超时

1. 返回可读超时提示，避免沉默。
2. 给出下一步建议（稍后重试或缩小问题范围）。

## 3. 幂等保护

- 同一任务轮次只允许一次“空回复重试”。
- 同一异常类型在同一轮次只触发一次恢复指令。
- 已输出最小回执后，不再重复输出同一模板。

## 4. 输出规范

- 优先短句，先结论后动作。
- 不编造未执行的工具结果。
- 若需要降级，明确说明“当前状态 + 下一检查点”。
