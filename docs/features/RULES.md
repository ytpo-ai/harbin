# Agent 功能文档维护规范

## 1. 功能文档层级结构

系统功能分为 **3级**：
- **1级（主要功能模块）**：如 `agent`（智能体）、`memo`（备忘录）、`meeting`（会议）
- **2级（具体功能）**：如 `agent/agent_mcp`（Agent MCP）、`memo/memo_mcp`（备忘录 MCP）
- **3级（子功能）**：2级功能下的细分功能

### 命名规范
```
1级/2级/3级
agent/memo/memo_mcp
meeting/orchestration/session
```

---

## 2. 功能文档规范

### 2.1 每个 2级功能 对应一个功能文档

- 路径：`docs/features/<1级>_<2级>.md`
- 示例：
  - `docs/features/agent_memo.md`（Agent + 备忘录）
  - `docs/features/meeting_orchestration.md`（会议 + 编排）
  - `docs/features/agent_skill.md`（Agent + 技能）

### 2.2 功能文档内容要求

每个功能文档应包含：

```markdown
# 功能名称

## 1. 功能设计
- 目标
- 数据结构
- 核心逻辑

## 2. 相关文档
- 规划文档 (docs/plan/)
- 开发总结 (docs/development/)
- 技术文档 (docs/technical/)
- API文档 (docs/api/)

## 3. 相关代码文件
- 后端代码
- 前端代码
```

### 2.3 文档依赖关系

功能文档 **引用** 而非重复以下文档内容：
| 文档类型 | 位置 | 说明 |
|----------|------|------|
| 规划文档 | `docs/plan/` | 需求背景、技术方案、里程碑 |
| 开发总结 | `docs/development/` | 实现细节、踩坑记录 |
| 技术文档 | `docs/technical/` | 底层设计、技术原理、数据模型设计(`docs/technical/db_schema/`) |
| API文档 | `docs/api/` | 接口定义 |

---

## 3. 开发规范

### 3.1 开发前必须阅读功能文档

**规则**：
1. 接到开发需求后，先找到对应的 **2级功能文档**
2. 如果功能文档存在且信息足够，则 **不必扫描代码文件**
3. 如果功能文档信息不足，再补充文档或查看代码

### 3.2 文档更新时机

| 操作 | 文档更新要求 |
|------|-------------|
| 新增功能 | 创建/更新功能文档 |
| 功能重构 | 更新功能文档 |
| API变更 | 更新功能文档 + API文档 |
| 技术优化 | 更新技术文档 |
