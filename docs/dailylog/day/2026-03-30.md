## Session 隔离后续修复：Skill 裁剪 + Prompt 阶段隔离 + Planner 纯文本 Retry

**完成事项**

- 修复 `phaseInitialize`/`phaseGenerate` 中 `plannerSessionId` 硬编码问题，改为统一使用 `withPlannerSession()` 方法写入 session state。
- 新增 `toolset-context.builder.ts` 中 `stripPhaseInitializeSectionIfNeeded()`：当 `roleInPlan` 以 `planner` 开头时，从 skill 文档中裁剪 `## phaseInitialize 行为` 段落，防止 system prompt 层级的指令误导 LLM。
- 在 4 个 planner 阶段的 user prompt 中注入阶段隔离声明：
  - generating（`planner.service.ts`）：声明当前阶段、禁止 phaseInitialize 指令、限定允许工具范围。
  - initialize（`planner.service.ts`）：反确认约束（"第一条回复必须是 tool_call"）。
  - pre_execute / post_execute（`orchestration-context.service.ts`）：阶段隔离声明。
- 新增 `agent-executor.service.ts` 中 `isPlannerTextOnlyRetryNeeded()`：当 planner 输出纯文本时注入系统纠正消息 retry 一次。
- 验证通过：generating 阶段 planner 正确调用 submit-task 生成任务；skill 裁剪生效；retry 机制正确触发。
