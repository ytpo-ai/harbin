- 模型管理 Agent 的 `systemPrompt/description` 也从通用 `agent-prompts.ts` 迁移到 seed 常量，确保“创建 model-management-agent”完全归属 seed。
- 会议总结链路改造为事件驱动：会议结束后发布 `meeting.ended` inner message，由 `meeting-assistant` 订阅并异步生成会议总结。
- 新增会议总结自动化服务，打通 `meeting-assistant` inbox 消费、消息处理状态回写与总结生成幂等保护。
- 补充本次改造 plan 文档并更新会议功能文档，确保“会后总结责任 Agent 与触发机制”可追溯。
- Inner Message 调整为 `senderAgentId` 可选：服务端缺省自动回填为 `system`，并同步放宽 schema 与接口参数约束。
- 完成“研发 Agent 双 runtime 路由恢复”改造：恢复 native 能力，并按任务类型在 `opencode/native` 间动态路由执行。
- Agents Runtime 新增任务路由判定优先级：`context.runtimeRouting.preferredChannel` > `agent.config.execution.taskRouting` > 默认任务类型映射。
- 编排异步任务链路补齐 `sessionContext.runtimeTaskType/runtimeChannelHint` 透传，确保 Agent Task Worker 可基于任务类型进行 runtime 选路。
- 完成 agents 应用构建验证（`pnpm -C backend run build:agents`）通过。
- 消息中心升级为“系统消息/内部消息/消息监听”三 Tab：消息监听支持全量 Agent 选择与名称/ID 搜索，配置 `inner-message-subscriptions` 覆盖计划编排/任务/会议状态事件（含 `meeting.ended`）并支持一键联动筛选内部消息。
