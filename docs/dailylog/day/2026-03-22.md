# 2026-03-22 工作日志

MCP Profile seed 职责迁移到 scripts，并移除系统角色重置权限集能力。

**完成事项**

- 新增 `backend/scripts/mcp-profile-seed.ts`，承载 MCP Profile seed 数据与落库逻辑。
- 调整 `backend/scripts/manual-seed.ts`，`mcp-profiles` 改为直接调用脚本实现，不再依赖 AgentService。
- 移除 service 层 seed 逻辑：`agent-mcp-profile.service.ts` 不再内置 seed 常量与 seed 写库方法。
- 删除 `resetToolPermissionSetsBySystemRoles` 后端接口链路（controller/service/role-service）。
- 删除前端对应调用与按钮（`frontend/src/services/agentService.ts`、`frontend/src/pages/Tools.tsx`）。
- 更新文档：plan、guide、feature、api。
- 新增 `backend/scripts/seed/builtin-tool-seed.ts`，将内置工具 seed 从 ToolService 迁移为独立脚本（支持 `sync/append` + `--dry-run`）。
- 新增 `backend/scripts/seed/seed-runner.ts` 并将 `seed:manual` 指向新入口，`builtin-tools` 不再依赖 Agents Nest 上下文。
- 新增 seed 兼容别名文件（`mcp-profile-seed.ts`、`system-schedule-seed.ts`、`role-seed.ts`、`seed-skill-planning-rules.ts`）保持迁移期路径稳定。
- 新增指南文档 `docs/guide/SCRIPTS_TOOLING_SEED_MAINTENANCE.MD` 沉淀脚本分层与运行边界。
- Agent Runtime 消息持久化改造：`agent_sessions` 从 `messages[]` 切换为 `messageIds[]` 引用式存储。
- `agent_messages` 扩展 step 级字段（`parentMessageId/modelID/providerID/finish/tokens/cost/stepIndex`），并新增相关索引。
- `agent_parts` 扩展 `step_start/step_finish`，Native 执行循环按 round 批量写入 message + parts。
- Runtime API 新增消息查询端点：session messages、message parts、run messages。
- AgentDetail Session 视图新增 step 元信息展示（finish/tokens/cost/stepIndex）并按 `parentMessageId` 分组。
- OpenCode 桥接补齐 step 持久化：按事件聚合 step 并落 `message + parts`，与 native 结构对齐。
- Model Service 升级为结构化返回（`response/usage/finishReason/cost`），并在会话层加入 usage 归一化处理。
- 修复运行时清理脚本入口：`cleanup:agents-runtime` 改为直连 `maintenance-runner`，并修复 runner 动态加载失败问题（改为 CommonJS `require`）。
- 新增模型定价服务 `model-pricing.service.ts`：接入 models.dev 远程价格、本地缓存、ModelRegistry 覆盖层与手动刷新能力。
- `ModelService.chat()` 接入统一计费收口：provider 未返回 `cost` 时基于 usage + pricing 自动补算。
- ModelRegistry Schema 与模型管理 CRUD 扩展 `cost` 字段，支持手动覆盖模型单价。
- 新增 Usage 模块（`usage.controller.ts`、`usage-aggregation.service.ts`、`usage.module.ts`）与 `agent_usage_daily_snapshots` 快照 Schema。
- 新增前端 `/usage` 页面与 `usageService`，提供概览卡片、趋势图、Agent/Model 排行和定价刷新操作。
- 新增研发指南 `docs/guide/USAGE_BILLING_SYSTEM.MD` 与功能文档 `docs/feature/USAGE_BILLING.md`，并补充 API 文档 Usage 章节。
- 修复 AgentSession 消息展示回归：Session 详情查询补齐 run 级 `user/system` 消息，并注入 `run.metadata.initialSystemMessages` 虚拟 system 消息，避免前端仅显示 assistant。
- 新增 fix 记录：`docs/issue/fix/2026-03-22-agentsession-system-user-display-regression.md`。
- AgentDetail Session 抽屉消息展示优化：隐藏 run/task/message 技术标识，新增“查看原始信息 + 复制”能力，消息正文改为完全折叠模型（system 默认折叠，其他默认展开）。
- 编排任务人工编辑后端能力落地：新增任务增删改、重排、批量更新、复制任务 6 个 API 端点，并补齐依赖合法性与循环依赖检测。
- `updateTaskDraft` 改为复用 `updateTaskFull` 核心更新逻辑，统一任务字段编辑通道并保持旧端点兼容。
- 前端 `orchestrationService` 新增人工编辑相关 API 方法与类型（`addTaskToPlan/deleteTask/updateTaskFull/reorderTasks/batchUpdateTasks/duplicateTask`）。
- `PlanDetail` 页面任务区完成基础可编辑化：新增“添加任务”弹窗、任务复制/删除、上下重排、批量保存（标题/描述/优先级/依赖）和脏状态提示。
- `Orchestration` 页面详情抽屉任务区同步完成基础可编辑化（与 PlanDetail 同步）：添加、复制、删除、上下重排、批量保存与脏状态提示。
- 更新编排功能文档与 legacy API 文档，补充新增人工编辑端点说明。
- Agent Prompt Catalog 元字段 `context` 重命名为 `slug`，并在 runtime system message 链路透传 `metadata.promptSlug`；前端 AgentDetail 系统消息标签改为优先显示 `promptSlug`。
- Agent Task Tool 级运行状态接入 Redis：`tool.pending/running/completed/failed` 写实时状态，任务终态统一回写 `idle`；`agent-master.list-agents` 支持 `agentId` 参数并返回 `runtimeStatus`。
- PlanDetail 页面完成 Run 历史信息架构：新增“任务设置/执行历史”双 Tab、最近一次 run 卡片、run 历史筛选列表与 run 详情抽屉（含 run task 明细）。
- Orchestration 页面计划详情抽屉补齐 Run 历史视图：新增“任务设置/执行历史”双 Tab、run 列表筛选及 run 详情抽屉。
- 编排 Plan 状态语义收敛为容器态（`draft/drafting/planned`）：移除运行终态承载，执行结果改由 Run 状态单独表达。
- 前后端新增 Plan 状态归一化兼容：历史 `running/paused/completed/failed` 旧值统一映射到容器态，避免界面继续展示“已完成”计划状态。
