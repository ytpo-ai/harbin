# 2026-04-05 工作日志

飞书 Bot 账号绑定升级为一次性 Token 方案，替换高风险邮箱直绑路径并补充前后端联动。

**完成事项**

- 新增后端接口 `POST /auth/me/feishu-bind-token`，签发 32 位随机 token，并写入 Redis（TTL 300 秒）。
- 新增签发限流：同账号默认 60 秒最多 3 次签发请求。
- Redis 基础设施新增 `getdel` 能力，支持 `GETDEL` 与 Lua 兼容回退，保证 token 原子一次性消费。
- 调整 `apps/channel` 的 `/bind` 逻辑，支持 `/bind token:<token>`，并在绑定成功后写入 `channel_user_mappings`。
- 邮箱绑定降级改为开关控制，默认关闭，仅管理员角色可使用（应急后门）。
- 前端 `Layout` 用户菜单新增「绑定飞书」入口，支持拉取 token、弹窗展示、复制绑定命令。
- 更新帮助文案与功能文档：`/help` 主指令改为 token 绑定，补充 `docs/feature/CHANNEL.md`。
- 新增计划文档：`docs/plan/CHANNEL_FEISHU_ONE_TIME_TOKEN_BIND_PLAN.MD`。

**影响范围**

- 后端认证模块（Auth API + Redis 令牌签发）。
- 渠道入站模块（飞书 `/bind` 指令解析与绑定流程）。
- 前端全局布局（用户下拉菜单与绑定弹窗交互）。
- 项目文档（Plan、Feature、Dailylog）。

**关联文档**
