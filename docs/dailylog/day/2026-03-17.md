- 重构 `ExecutorSelectionService`：从关键词匹配升级为多维能力路由（角色/工具/能力/关键词），并支持 `lock_to_planner` 分配策略。
- 修复 Orchestration 分配链路：计划创建与 replan 都传入 `plannerAgentId + assignmentPolicy`，支持强约束场景下任务归属锁定。
- 修复 Skill 激活链路：`agent.service` 引入按需激活规则，命中时加载 skill content 注入 prompt；默认仍走元数据轻量注入。
- 修复 Prompt 重复注入：若任务描述已出现在历史 user 消息中，则不再重复注入完整 description，降低 token 浪费。
- 新增 `send_internal_message` 内置工具：Agent 可调用 legacy `/inner-messages/direct` 真实发送内部消息并拿到 `messageId` 回执；同时补齐工具 seed 与 API/功能文档。
- 将 `send_internal_message` 加入 MCP Profile 系统种子：默认下发到 `executive-lead/management-assistant/technical-architect/fullstack-engineer/system-builtin-agent/meeting-assistant` 角色工具集。
- 新增 `seed:manual --mode=append`：支持仅追加新工具/新 profile，避免覆盖既有工具元数据与 profile 手工配置（默认仍为 `sync`）。
- 修复 append seed 运行期冲突：移除 MCP Profile seed 对 `tools` 的冲突更新路径（`$setOnInsert.tools` + `$addToSet.tools`），并在 manual-seed 执行时禁用 `AgentTaskWorker`，避免退出阶段 Redis 连接关闭报错。
- 执行会议上下文优化 Step1：`任务信息/身份与职责` 改为 fingerprint+delta 注入；会议场景新增“一次确认后执行 + 三段式回执 + 短版通知 + 1轮闭环SLA”执行规则；空响应自动重试并兜底“操作进行中，1 分钟内补充回执。”。
- 修正 Step1 会话噪音细节：会议场景下跳过重复 `systemPrompt` 与固定 `任务信息`，并将历史 system 上下文前置，优化 session 可读性与顺序稳定性。
- 调整 runtime 首轮落盘顺序：先写入构建后的 system 上下文，再写 user 触发消息，便于 session 排查与阅读。
- 精简工具可用清单注入：从 JSON（含参数）改为 `name | description` 列表，降低会议会话上下文体积。
- 修复 Identity 备忘录聚合输出：移除“工具描述”章节，工具集统一改为 ID 列表展示并补齐去重测试。
