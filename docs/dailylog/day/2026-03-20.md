# 2026-03-20 工作日志

**完成事项**

- 新增 Skill 管理详情页「Agent 绑定」Tab 的解除绑定能力，支持对已绑定 Agent 发起解绑。
- 解绑交互增加二次确认与提交中态，避免误操作与重复提交。
- 解绑请求复用 `/skills/assign` 接口并透传 `enabled=false` 语义。
- 绑定/解绑完成后统一刷新 `all-skill-agents` 与目标 Agent 技能缓存，保持列表与抽屉状态一致。
- 更新 Agent Skill 功能文档，补充前端绑定/解绑交互约定。
- 完成 Agent Action Logs 迁移到 `apps/agents`：新增 schema/service/controller/module，保留 `sourceEventId` 幂等索引与 runtime 事件映射查询能力。
- Runtime 日志同步链路从跨服务 HTTP 调用 legacy `3001` 改为 agents 进程内 service 调用，减少跨服务依赖与失败面。
- Gateway 路由更新：`/api/agent-action-logs/**` 统一转发到 agents 服务（3002）。
- 更新 Agent Runtime 与 API 文档，标注 action logs 能力归属已迁移至 agents。
- 收敛 Agent Action 产生日志入口到 agents：`/agents/:id/execute` 与 `/tools/:id/execute` 在 agents 内直接记录 action log（started/completed/failed）。
- legacy `AgentClientService` 移除主动写 action log 逻辑，改为仅透传 `executionContext` 给 agents 侧执行链路。
- 完成 Inner Message 迁移至 agents：新增 `apps/agents` 内消息模块（controller/service/dispatcher/runtime-bridge/collaboration-automation）。
- legacy 编排与会议链路改为通过 `AgentClientService` 调用 agents 侧 `/inner-messages/publish`，移除对 legacy `InnerMessageModule` 的运行时依赖。
- 内置工具 `send_internal_message` 调用路由切换到 agents 服务地址，避免跨服务回环依赖 legacy 入口。
- 更新 Inner Message 功能与 API 文档，补充迁移计划文档与服务边界说明。
- 修复 Gateway 路由漏配：`/api/inner-messages/**` 与 `/api/inner-message-subscriptions/**` 统一转发到 agents 服务（3002）。
- 完成 Agent-role 主数据迁移到 agents：新增 `apps/agents` 侧 `agent-role` schema，并将 `/agents/roles*` 从代理模式切换为本地读写（含 CRUD）。
- Agent 角色校验与工具权限链路去除 legacy `/roles` 运行时依赖：`AgentRoleService` 与 `ToolService` 改为直接读取 agents 侧 `agent_roles`。
- HR 前端角色管理 CRUD 接口切换到 `/agents/roles*`，保持页面角色管理与 Agent 创建角色来源一致。
- 新增迁移计划文档并更新 API/功能文档，明确角色主数据归属已迁移到 agents。
- Agent 详情页 Session 抽屉完成消息展示优化：正文默认 5 行折叠、支持展开/收起、按条展示 parts 数量并可展开查看 parts 内容。
- Session 抽屉头部新增刷新图标按钮，支持手动刷新当前 session 详情及列表数据。
- Agent Session 消息轨迹交互优化：系统消息不再展示 parts 标记；`Parts` 数量与查看入口合并为单一按钮；part 内容默认折叠 5 行并支持展开。
- 实现 EI 文档热度统计链路：新增 `ei_doc_commit_facts`/`ei_app_configs` schema、配置服务与热度核心服务，支持 Redis + Mongo 双层存储与 8H/1D/7D 排行回填。
- 新增 EI API：`/ei/docs-heat/*` 与 `/ei/config/docs-heat`，支持热度触发、排行查询、latest 状态和权重配置更新。
- Agents 工具链新增 `builtin.sys-mg.mcp.rd-intelligence.docs-heat-run`，并接入 internal API client 执行 EI 文档热度刷新。
- 编排调度新增系统计划 `system-docs-heat`：seed 支持幂等创建，Scheduler 暴露 `GET/POST /orchestration/schedules/system/docs-heat*`。
- 前端工程统计页升级为双 Tab（工程规模统计/文档热度），新增文档热度排行表、窗口切换、触发统计、权重配置弹窗。
- 更新 EI API 文档，补充文档热度与配置接口说明。
