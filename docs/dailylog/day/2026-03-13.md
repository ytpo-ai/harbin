# 2026-03-13 工作日志

**完成事项**

- 完成研发会话页交互优化：新增“先选择研发 Agent，再同步 OpenCode projects”的操作流。
- 后端复用 `rdproject` 并调整集合名为 `ei_projects`，新增同步来源字段与幂等索引。
- 新增 `POST /rd-management/agents/:agentId/opencode/projects/sync` 接口，按 Agent 维度批量同步并返回统计结果。
- 限制项目创建入口：`createProject` 改为仅允许同步链路创建，禁止前端手工创建 EI 项目。
- 前端 `RdConversation` 页面接入 Agent 列表筛选（`execution.provider=opencode`）和同步按钮，项目列表按 Agent 过滤。
- 二次收敛研发会话页面交互：移除前端创建 Session 与本地绑定入口，顶部统一为 Agent 选择 + EI Project 选择 + 同步按钮。
- 修复 Agent 项目同步筛选：按 `endpointRef` 拉取 OpenCode 项目并修正 scope 匹配规则，避免根路径 `/`（global）误命中。
- 修复 OpenCode sessions 拉取逻辑：同步链路改为按 `directory` 查询会话，避免把 global sessions 写入项目元数据；并支持 `/rd-management/opencode/sessions?directory=...`。
- 完成 OpenCode 后端去 SDK 化：`agents` 与 `rd-management` 统一改为 OpenCode HTTP API 直连（含 SSE 事件流），移除 `@opencode-ai/sdk` 依赖。
- 修复 OpenCode session 模型对齐：session 创建链路补充 `model(providerID/modelID)` 透传，确保会话模型与 Agent 绑定模型一致。
- 研发会话页新增 Agent 模型展示与“新建 Session”入口，创建后自动刷新会话/上下文。
- 修正研发会话 events 视图同步范围：按当前选中 `sessionId` 过滤事件，避免跨会话混入。
- 研发会话发送消息链路改为默认携带当前选中 Agent 的 `model(providerID/modelID)`，避免会话级默认模型覆盖执行模型。
- 增加 OpenCode 模型能力校验：发送/创建会话前尝试读取模型目录，若不支持 Agent 指定模型则返回明确报错，并在前端弹窗提示。
- 修复模型热恢复误报：当出现“模型不支持”时，后端会强制刷新模型目录并自动重试一次请求，避免 OpenCode 刚恢复时因缓存/瞬时状态继续报错。
- 新增工程统计能力：EI 后端新增统计快照模型与 `statistics/snapshots` 系列接口，支持 docs/frontend/backend 项目明细与汇总落库。
- 前端在 `研发智能` 下新增 `工程统计` 二级菜单，接入统计按钮、汇总卡片、明细表与历史记录。
- Agents 工具新增 `builtin.sys-mg.mcp.rd-intelligence.engineering-statistics-run`，支持 Agent 触发同一统计链路。
- 工程统计接入系统编排：新增系统默认计划 `system-engineering-statistics`，按钮触发改为触发该计划一次执行。
- 调度器支持基于 `schedule.input.payload.toolId` 执行 MCP 工具路径，工程统计计划通过工具执行并由消息中心通知前端。
- 完成消息中心模块首版（legacy 落地）：新增 `message-center` 后端模块、`system_messages` 模型、消息列表/未读数/已读接口与工程统计 Hook 写入。
- 前端完成全局 Header 右侧消息入口与用户区，新增消息抽屉与完整消息中心页，支持点击消息标记已读并跳转到工程统计上下文。
- 修复工程统计 backend 口径：补齐 `backend/apps/<app>` 各子应用独立统计行，汇总纳入 `totalBackendBytes`。
- 修复工程统计消息通知链路：前端按钮触发系统计划时默认注入当前用户 `receiverId`，并透传到 MCP 工具参数，保证统计完成后消息中心可见。
- 新增研发智能“需求管理 + 智能研发看板”首版：支持需求创建、列表筛选、详情讨论、分发给研发 Agent、状态流转与看板聚合。
- 打通 GitHub Issues 一键同步：需求详情/列表可触发同步并回写 Issue 映射信息。
- 加固前端触发参数：`工程统计` 按钮调用调度接口时显式传 `receiverId=currentUser.id`，避免后端回填在边界场景失效。
- 修复消息中心 Header 红点同步：消息已读后广播 `message-center:updated` 事件，Header 立即刷新未读数并联动抽屉列表。
- 加固消息已读一致性：已读成功后补拉未读数并随事件广播，消息中心页增加本地未读回显，消除红点滞后。
- 调整消息抽屉交互：点击铃铛打开即请求未读消息列表；抽屉消息点击进入消息中心详情并自动已读；新增“全部已读”按钮。
- 文案优化：右侧消息抽屉单条操作按钮由“已读”改为“标记已读”。
- 工程统计页交互升级：历史快照改为列表主视图，新增状态筛选、分页与“查看快照”详情抽屉（汇总/项目明细/异常信息），并支持 `snapshotId` 深链打开。
- 新增 EI 项目三源模型：`local/opencode/github`，并引入项目绑定关系（1 local -> N opencode + 1 github）。
- GitHub 仓库绑定改为复用 API Key：项目仅保存 `githubApiKeyId` 引用，不落库存储明文 token。
- RD Management 新增项目接口：`POST /projects/local`、`POST /projects/bind/opencode`、`POST /projects/bind/github`。
