# 研发日常工作进度记录

## 1. 文档结构

按周和天为周期，聚合记录每天的工作内容、遇到的问题、解决方案和后续计划。文档路径示例：

```docs/dailylog/day/
├── 2024-06-01.md
├── 2024-06-02.md
├── 2024-06-03.md
└── ...
docs/dailylog/week/
├── 2024-W22.md
├── 2024-W23.md
└── ...
```

每篇日记文档包含以下结构：

```markdown# 

YYYY-MM-DD 工作日志

内容摘要，如`新增 RD 文档写入 MCP 工具`

**完成事项**


**影响范围**


**关联文档**

```

## 2. 日常进度记录规范

### 内容要求
- 日报尽量详细描述
- 周报相对日报可以更聚合和总结

### 工作分以下几种
- 新功能开发，（必须记录）。
- 现有功能优化（建议记录）。
- Bug 修复 (选择性记录)。


## 3. 约束执行检查点

在 opencode 完成功能开发后，必须检查：

- [ ] 是否需要记录日常进度？如果需要，是否已添加相关工作内容到 `docs/dailylog/` 下的 Markdown 文档
