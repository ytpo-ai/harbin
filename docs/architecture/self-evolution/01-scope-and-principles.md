# 分层 1：范围与原则

## 1. 目标定义

自我进化系统目标：
- 在给定约束下提升任务成功率、效率、成本收益与协作质量
- 自动提出改进方案并通过实验验证
- 将有效改进沉淀为可复用策略

## 2. 范围（In Scope）

- 基于运行数据感知系统能力
- 将架构师预期转为机器可执行目标（Intent）
- 自动生成并管理改进提案
- 对改进方案进行 A/B 或灰度验证
- 基于价值函数进行推广、暂停或回滚决策

## 3. 非目标（Out of Scope）

- 不做无边界自主改代码
- 不在无审批场景下直接执行高风险变更
- 不以单一指标作为“进化成功”的唯一依据

## 4. 基本原则

1. 可测量优先：没有指标不允许自动优化
2. 小步快跑：低风险先自动，高风险必须审批
3. 可回滚：每个实验与策略变更都要有回滚点
4. 守红线：稳定性、安全、合规优先于局部收益
5. 证据驱动：所有推广决策必须附带实验证据

## 5. 角色职责

- 架构师
  - 设定 Intent（目标、约束、优先级）
  - 审批高风险提案与推广决策
- 系统（进化控制面）
  - 自动感知、分析、提案、实验、评估
- 人类运营/治理角色
  - 查看证据包，做最终授权与复盘

## 6. 成功标准

建议设置季度级目标：
- 任务完成成功率提升 >= 10%
- 单任务平均成本下降 >= 15%
- `waiting_human` 比例下降 >= 20%
- 失败计划比例下降 >= 20%
- 严重回归事故数 = 0
