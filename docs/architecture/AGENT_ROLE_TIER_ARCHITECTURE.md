# Agent 角色分层架构（高管/执行层/临时工）

## 1. 目标与范围

- 建立一套对 Agent 与人类员工通用的角色分层模型。
- 统一定义各层默认能力、任务分派关系与治理边界。
- 为后续权限系统、调度编排、Skill 注入提供单一规则来源。

## 2. 角色分层模型

系统角色统一分为三个级别：

1. **高管层（Leadership）**
2. **执行层（Operations）**
3. **临时工（Temporary Worker）**

该分层同时适用于：

- Agent 角色
- 人类员工角色

## 3. 各层角色定义

### 3.1 高管层

包含以下角色：

- CTO 角色
- CEO 角色
- 与高管层级人类员工绑定的专属助理

### 3.2 执行层

除高管层与临时工之外的所有角色。

### 3.3 临时工

仅包含临时工角色的 Agent。

## 4. 能力基线与权限边界

### 4.1 高管层默认能力

高管层默认具备以下能力：

- 会议创建能力
- 计划编排的全部能力
- 网络检索能力
- 可根据任务类型自由选择自身执行模型

说明：

- 上述能力为高管层基础能力，不依赖具体岗位二次授权。
- 模型自由选择仅针对高管级别 Agent 的任务执行阶段，不改变平台安全与合规约束。

### 4.2 执行层默认边界

- 执行层不自动继承高管层全部治理能力。
- 执行层可接收并执行高管分派的子任务。
- 执行层可将可拆分工作继续分派给临时工 Agent（若任务链路允许）。

### 4.3 临时工默认边界

- 临时工默认不具备任何系统管理工具。
- 临时工仅能完成由高管层或执行层 Agent 分配的子任务。
- 临时工不承担计划编排与全局治理职责。

## 5. 分派与协作链路

### 5.1 标准分派方向

- 高管 -> 执行层
- 高管 -> 临时工
- 执行层 -> 临时工

### 5.2 治理责任

- 高管对拆解后的子任务负责总体目标与优先级治理。
- 执行层对具体交付结果负责。
- 临时工对被分配任务的局部执行质量负责。

## 6. 指挥权与冲突仲裁

### 6.1 高管内部优先级规则

- 人类高管的专属助理代表该人类高管意志。
- 在授权范围内，专属助理指令优先级高于其他普通 Agent 的同级建议。
- 在特定场景中，其他高管需遵守人类高管专属助理给出的指令。

### 6.2 特定场景（强约束）

“其他高管必须遵守”仅在以下场景生效：

- 明确标注为人类高管直接授权的任务。
- 涉及跨团队优先级调整、计划冻结、资源再分配的治理动作。
- 指令未与平台安全策略、合规限制冲突。

### 6.3 冲突处理顺序

当多高管指令冲突时，按以下顺序处理：

1. 安全与合规约束
2. 人类高管专属助理的授权指令
3. 既定计划编排规则
4. 常规高管协商结论

## 7. 架构落地建议

- 在角色模型中新增统一字段 `tier`，枚举值：`leadership | operations | temporary`。
- 在工具授权链路增加“层级默认策略 + 角色增量策略”合并机制。
- 在编排系统中显式记录任务来源层级与分派路径，支持审计回放。
- 对“专属助理高优先级指令”增加授权标记，避免被误用为通用越权。
