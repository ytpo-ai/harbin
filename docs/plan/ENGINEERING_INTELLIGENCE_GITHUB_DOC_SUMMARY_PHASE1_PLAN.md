# 研发智能 GitHub 文档阅读与摘要（Phase 1）计划

## 需求目标

实现研发智能应用第一阶段能力：

1. 支持在页面配置研发可访问的 GitHub 仓库 URL。
2. Agent 可读取仓库文档（README、docs 等）并生成摘要。
3. 输出结构化结果（仓库级摘要 + 分文档摘要 + 证据片段）。

## 执行步骤

1. 设计数据模型与接口：新增“仓库配置”实体，定义新增仓库、查询仓库、触发摘要的 API。
2. 实现 GitHub 文档采集能力：基于 `GITHUB_TOKEN` 拉取目标仓库文档清单与正文，支持 `README*`、`docs/**`、`CONTRIBUTING*`、`ARCHITECTURE*`、`ADR*`。
3. 实现摘要流水线（MVP）：完成单文档摘要与仓库聚合摘要，输出关键信息、技术信号与证据引用。
4. 新增前端研发智能页面：支持配置仓库 URL/分支、查看仓库列表、手动触发摘要并查看结果。
5. 联调与验证：完成基础错误处理（权限不足/仓库不存在/无文档），并运行 lint 与构建验证。
6. 文档更新：补充 README 与 API 文档，说明 `GITHUB_TOKEN` 和 CTO 文档摘要入口。

## 关键影响点

- 后端：新增模块（controller/service/schema/dto），新增 GitHub 文档采集与摘要逻辑。
- API：新增研发智能文档摘要相关接口。
- 前端：新增仓库 URL 配置与摘要查看页面。
- 测试/验证：至少完成 lint + build 级别检查，确保主流程可用。
- 文档：更新使用说明与 API 文档。

## 风险与依赖

- 依赖 `GITHUB_TOKEN` 对目标仓库具备读取权限（私有仓库必需）。
- 不同仓库文档结构差异大，首版摘要准确性受文档质量影响。
- 大仓库文档数量较多时，采集与摘要耗时可能上升，需限制单次文件数与内容长度。

## Phase 1.1 增强（文档浏览与更新追踪）

### 新增目标

1. 按仓库目录结构展示 `docs/` 文档树。
2. 点击文档可在线浏览内容。
3. 可查看文档更新记录与辅助信息（提交历史、作者、最近更新时间等）。

### 新增执行步骤

1. 后端新增文档树接口：返回 `docs/` 递归目录结构（file/dir）。
2. 后端新增文档详情接口：返回 markdown 原文与文件元信息。
3. 后端新增文档历史接口：返回文档最近 commit 记录与贡献者统计。
4. 前端改造研发智能页面：左侧目录树、中间文档阅读、右侧历史与辅助信息。
5. 更新 API/README 文档并完成前后端构建验证。

### 新增影响点

- 后端：`engineering-intelligence` 模块接口扩展。
- 前端：研发智能页面信息架构升级（文档浏览工作台）。
- 文档：API 与使用说明新增“文档浏览与历史追踪”能力说明。

## Phase 1.2 修复（文档抽屉阅读）

### 修复目标

1. 修复点击文档后未稳定展示正文的问题。
2. 将文档阅读区调整为右侧抽屉展示。

### 修复步骤

1. 增强前端文档加载态与错误态，明确展示加载失败原因。
2. 文档点击后改为打开右侧抽屉，抽屉内展示正文、路径与 GitHub 链接。
3. 保留主区域目录树和历史面板，抽屉关闭后不影响浏览上下文。
