# OpenCode 长任务抗超时改造计划（Worker + SSE）

## 1. 背景与问题

- 当前“创建 opencode session 后执行推理”在长时任务场景中容易超时。
- 现有同步请求链路将“任务执行生命周期”与“HTTP 连接生命周期”耦合，导致连接中断即任务感知中断。
- 仅靠前端重连无法解决执行层中断、网关超时、事件丢失与状态不一致问题。

## 2. 目标与范围

### 2.1 目标

1. 将任务执行改造为异步 Worker 模型，彻底解耦请求与执行生命周期。
2. 通过 SSE 提供稳定实时事件流，支持断线重连与事件续传。
3. 建立任务状态机、取消/超时/重试机制，保证可恢复与可观测。
4. 保持前端在主应用 `frontend/` 内完成接入，不新增独立前端工程。

### 2.2 范围

1. 后端：任务 API、Worker、事件持久化、SSE 推送。
2. 前端：任务创建、SSE 订阅、重连与续传、失败兜底展示。
3. 运维：网关/代理超时与 buffering 配置、监控告警、灰度开关。

### 2.3 非目标

1. 本期不做分布式 run 锁重构。
2. 本期不扩展到全部执行引擎，仅聚焦 OpenCode 执行通道。

## 3. 执行步骤（按顺序）

1. 明确契约与状态模型
   - 定义任务状态机：`queued -> running -> succeeded|failed|cancelled`。
   - 定义统一事件 envelope 与事件类型（`status/progress/token/log/result/error/heartbeat`）。
   - 关键影响点：后端、API、前端。

2. 落地异步任务执行模型
   - 新增任务创建接口，快速返回 `taskId`。
   - 引入队列与 Worker 执行 OpenCode 推理，执行过程中持续产出事件。
   - 关键影响点：后端、任务调度、稳定性。

3. 落地 SSE 推送与续传
   - 提供 `GET /tasks/:taskId/events` SSE 接口。
   - 支持 `id` / `Last-Event-ID` 续传，断线后补发缺失事件。
   - 关键影响点：后端、前端、网关配置。

4. 增加取消/超时/重试与幂等
   - 支持任务取消、总超时、步骤超时、可重试错误退避重试。
   - 增加 `idempotencyKey` 防重提交，保证重复请求不重复执行。
   - 关键影响点：后端、数据一致性。

5. 完成前端接入与降级策略
   - 创建任务后进入任务详情态并订阅 SSE。
   - SSE 连续失败后降级轮询 `GET /tasks/:taskId`。
   - 关键影响点：前端、用户体验。

6. 完成运维与可观测建设
   - 配置代理超时与禁用响应缓冲。
   - 输出任务与连接核心指标，配置告警阈值。
   - 关键影响点：运维、稳定性。

7. 灰度上线与验收
   - 通过开关逐步放量，验证 20 分钟长任务稳定性与断线恢复。
   - 关键影响点：发布流程、质量保障。

## 4. 关键设计决策

1. 前后端采用 SSE（非 WebSocket）作为默认任务事件通道。
2. Worker 与 OpenCode 交互优先采用 CLI 子进程流式协议（JSONL），后续可替换 SDK 适配器。
3. 事件先落库后推送，确保可回放、可补偿、可审计。
4. 心跳采用“服务端主动发心跳 + 客户端超时重连”双侧保活。

## 5. 风险与依赖

1. 反向代理默认超时与 buffering 会导致 SSE 假在线或消息延迟。
2. 多实例下事件顺序与补发一致性依赖事件索引与唯一约束设计。
3. 取消语义要与 OpenCode 执行器中断行为统一，否则会出现“状态已取消但进程仍运行”。
4. 长任务事件量较大，需控制存储保留周期与 payload 大小。

## 6. 验收标准

1. 单任务稳定执行 20 分钟以上，前端持续收到事件且无请求超时中断。
2. 断网 30 秒后恢复连接，能够基于 `Last-Event-ID` 补齐事件。
3. 页面刷新后可恢复任务状态与进度，不出现重复渲染或顺序错乱。
4. 取消请求在可接受时延内生效（目标 < 5 秒）。
5. 关键指标可观测：任务成功率、P95 耗时、SSE 断连率、重连次数。

## 7. 实施里程碑

### M1（MVP）

- 异步任务 API + Worker 执行 + SSE 推送 + 心跳。

### M2（增强）

- 续传补发、取消、重试、幂等、前端降级策略。

### M3（生产化）

- 监控告警、容量治理、灰度放量、运维手册与故障演练。

## 8. 受影响模块

### 后端

- `apps/agents`：任务控制器、任务服务、Worker、OpenCode 适配器、事件持久化。
- `apps/gateway`：SSE 相关路径透传与超时配置校验。

### 前端

- `frontend/`：任务详情页、SSE 订阅 Hook、断线重连与轮询兜底。

### 数据与运维

- Mongo 任务与事件集合、索引、TTL。
- 代理与网关配置、告警面板。
