# RD Management EI Project Sync Plan

## 1. 背景与目标

- 优化研发管理页面交互：用户需先选择研发 Agent，再同步其关联的 OpenCode projects。
- 后端新增/调整项目存储模型，集合名统一为 `ei_projects`。
- 项目记录只能通过同步链路创建，前端不再提供手动创建入口。

## 2. 执行步骤

1. 梳理现状与复用点：明确当前研发管理页、`rdproject` 模型、OpenCode 项目来源接口。
2. 后端模型重构：基于现有 `rdproject` 调整为 `ei_projects`，补充字段和唯一约束。
3. 同步接口实现：新增按 `agentId` 同步 OpenCode projects 的接口，支持幂等写入和统计返回。
4. 创建入口约束：移除/禁用前端创建入口，并在后端限制仅同步链路可创建。
5. 前端交互改造：先选研发 Agent，再触发同步并刷新项目列表。
6. 质量与文档：完成联调、测试与文档更新（feature/api/dailylog）。

## 3. 关键影响点

- 后端：数据模型、服务层同步逻辑、权限与数据约束。
- API：同步接口与项目查询返回结构。
- 前端：研发管理页面交互流程与入口调整。
- 数据库：集合名落库 `ei_projects`，兼容历史 `rdproject` 数据。
- 测试/文档：回归测试、接口文档和功能文档更新。

## 4. 风险与依赖

- OpenCode 项目来源接口字段稳定性会影响映射逻辑。
- 历史数据兼容策略需要避免破坏现有查询能力。
- 并发同步下需保证幂等与去重一致性。

## 5. 完成标准

- 研发管理页面满足“先选 Agent，再同步项目”。
- `ei_projects` 模型与接口生效，项目仅可通过同步创建。
- 同步接口可重复调用且结果稳定（无重复脏数据）。
- 文档与日志按规范更新。

## 6. 2026-03-13 交互收敛（二次调整）

1. 删除前端“创建 Session”能力（输入区、按钮、mutation 与调用入口）。
2. 将 Agent 选择与 Project 选择统一上移到页面顶部主操作栏，Project 选择位于 Agent 选择右侧。
3. 删除“本地项目绑定/EI项目绑定”前端能力（下拉、按钮、绑定接口调用入口）。
4. 保留“选 Agent -> 同步 Projects -> 选 Project”主流程，保持消息查看/发送与事件面板能力。
