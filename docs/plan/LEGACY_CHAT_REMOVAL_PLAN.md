# 废弃协作功能移除计划

## 背景

- 当前项目确认不再保留已废弃的协作功能。
- 目标是删除相关代码与文档记录，避免前后端与文档继续出现该能力。

## 执行步骤

1. 全量扫描 `frontend/`、`backend/`、`docs/` 中相关实现与引用，确定删除清单。
2. 删除前端代码（页面、服务、类型与路由/菜单引用），并清理关联调用。
3. 删除后端代码（controller/service/schema 及模块注册），并移除任务编排中的对应依赖分支。
4. 清理共享类型与消息模型中的字段枚举与接口定义，保证类型系统一致。
5. 清理 `docs/` 下相关记录，移除 API、WS、架构及计划/开发文档中的对应描述。
6. 运行 lint/typecheck（按项目脚本）验证删除后代码可编译、可通过静态检查。

## 影响范围

- 前端：路由、导航、页面与服务、共享类型。
- 后端：chat 模块、tasks 协作模式、共享 schema/types。
- 文档：API/架构/计划/开发文档中的相关记录。

## 风险与处理

- 风险：移除逻辑可能与会议或编排流程的上下文逻辑有交叉。
- 处理：删除前先做引用扫描，删除后跑 lint/typecheck，按报错补齐清理。
