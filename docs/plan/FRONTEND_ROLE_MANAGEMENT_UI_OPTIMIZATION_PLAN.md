# Frontend Role Management UI Optimization Plan

## 1. 目标与背景

- 优化角色管理页面的信息层级，突出“角色管理”为主任务。
- 页面默认展示角色列表，创建角色改为弹窗完成，减少页面跳转和上下文切换。
- 移除页面中的“员工绩效评估”部分，降低非目标信息干扰。

## 2. 执行范围

### 2.1 In Scope

- 调整角色管理页面标题与副标题文案。
- 将“创建角色”入口改为弹窗交互，并保留创建后刷新列表能力。
- 确保角色列表为页面默认可见内容。
- 移除当前页面“员工绩效评估”相关 UI 区块。

### 2.2 Out of Scope

- 不调整后端角色管理 API 契约。
- 不新增独立前端工程或迁移页面所在模块。
- 不修改与角色管理无关的全局导航策略。

## 3. 执行步骤

1. 识别 `HRManagement` 页面当前结构，确认标题、列表、创建流程、绩效区块位置。
2. 优化标题文案与视觉层级，保持现有设计体系一致。
3. 调整为默认展示角色列表，并将创建角色改为弹窗实现。
4. 移除员工绩效评估区块及其在页面内的相关入口。
5. 回归验证：列表首屏、弹窗创建链路、页面渲染与状态切换稳定。
6. 执行前端 lint/build 检查并修复阻塞问题。

## 4. 关键影响点

- 前端页面布局与交互（标题、列表、弹窗、区块结构）。
- 前端状态管理（弹窗开关、表单提交后刷新列表）。
- 测试与构建（lint/build）。
- 文档（计划文档落盘）。

## 5. 风险与依赖

- 若绩效区块与页面状态共享较多，需要同步清理关联状态，避免空状态异常。
- 若创建角色逻辑依赖内联表单，改为弹窗后需验证表单重置与错误提示行为。
