# 会议系统提示干扰治理计划

## 需求理解

- 当前会议场景中出现“系统推断意图提示”与“用户明确动作请求”冲突，导致助理被无关提示（如模型列表）干扰。
- 本次目标是定位系统提示写入时机与触发条件，并建立可执行的优先级裁剪机制，确保助理优先完成用户当前显式任务。

## 执行步骤

1. 梳理会议消息到提示构建的完整链路，定位系统提示注入入口（任务信息、工具策略、意图提示）。
2. 复盘冲突样本（记事请求被模型列表提示覆盖），明确触发规则、命中条件与执行优先级。
3. 设计冲突裁剪规则：用户显式动作意图优先于系统推断意图；仅同域提示可保留并增强。
4. 在提示拼装层实现冲突检测与过滤，冲突时降级/屏蔽低优先级提示。
5. 补充回归测试，覆盖“记录请求优先”与“真实模型列表查询”两类路径。
6. 更新功能文档，沉淀触发条件矩阵与优先级规范。

## 本轮追加范围（2026-03-10）

1. 优化 `resolveMentionedAgentIds`：移除短前缀模糊命中，仅保留精确别名匹配与安全归一化匹配。
2. 删除其余意图识别中的宽匹配关键词分支（如仅靠“当前/现在/日志”等弱信号触发）。
3. 联动更新路由条件与文档说明，确保显式意图优先且可解释。

## 本轮追加范围（二）（2026-03-10）

1. 显式短语命中升级为方括号命令模式：仅当短语以 `[短语]` 或 `【短语】` 形式出现时触发。
2. 短语库精简为“每项能力中文 1 条 + 英文 1 条”。
3. 在 `docs/technical/` 记录完整短语清单、命中规则与示例。
4. 会议聊天输入框支持输入 `[` 或 `【` 时弹出短语建议，点击后自动填充。

## 本轮追加范围（三）（2026-03-10）

1. 修复 `agent.service.ts` 会议编排意图误判：收窄触发词，移除“执行/继续/开始”等单独触发。
2. 增加编排否定识别（如“不要执行计划/不是编排”），命中后阻断编排守卫。
3. 统一编排工具 ID 到 canonical（`builtin.sys-mg.mcp.orchestration.*`），并兼容 legacy `mcp.orchestration.*` 映射。
4. 排查并修复同类 canonical/legacy 混用问题（含模型管理与 human operation log 工具 ID 判定）。

## 关键影响点

- 后端：会议任务上下文构建、系统提示注入、意图路由策略。
- 测试：提示合并与路由优先级回归测试。
- 文档：`docs/feature/MEETING_CHAT.md`（必要时补充规则说明）。

## 风险与依赖

- 系统提示来源可能分散在多层（编排任务、提示模板、策略注入），需要统一优先级口径。
- 过度过滤可能影响本应触发的系统守护提示（如安全/权限），需保留高优先级白名单。

## 验证方式

- 输入“请记录/追加备忘录”类请求时，不应触发模型列表策略提示。
- 输入“系统里有哪些模型/模型列表”时，应正确触发模型列表工具提示并执行。
- 冲突场景下日志可观察到提示过滤决策（命中规则、保留/屏蔽结果）。
