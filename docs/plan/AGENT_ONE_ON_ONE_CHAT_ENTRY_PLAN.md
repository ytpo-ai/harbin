# Agent 一对一聊天入口计划

## 需求目标

支持会议兼容 `1 对 1` 模式，并在 Agent 入口增加“开始聊天”按钮，点击后直接进入与该 Agent 的聊天会话。

## 执行步骤

1. 梳理现有 Agent 列表、会议/聊天页面路由与会话创建流程，确认一对一会话的复用与创建规则。
2. 在 Agent 列表卡片（或对应入口区域）新增“开始聊天”按钮，并接入统一的点击处理逻辑。
3. 实现“直达聊天”能力：优先复用当前用户与目标 Agent 的一对一会话；若不存在则自动创建会话。
4. 进入会话时显式标记或约束为 `1 对 1` 模式，避免误入群组会议流程。
5. 完成前后端联调与页面状态处理（加载态、错误提示、跳转成功反馈）。
6. 补充/更新相关测试与文档，确保功能可回归验证。

## 关键影响点

- 前端：Agent 页面交互、会话跳转、状态管理。
- 后端/API：会话查询/创建接口的一对一模式兼容性。
- 数据库：会话模型是否支持按 `owner + agent + mode` 唯一定位。
- 测试：新增按钮交互与会话复用/创建路径验证。
- 文档：更新使用说明和相关 API/功能文档。

## 风险与依赖

- 现有接口若无一对一会话复用能力，需新增查询条件或幂等创建逻辑。
- 聊天与会议页面参数约定若不一致，可能导致跳转后展示异常。
- 历史会话数据可能缺少模式标记，需兼容旧数据读取策略。
