# Skill 详情页 Agent 绑定勾选化与全选优化计划

## 背景

- 当前 Skill 详情页「Agent 绑定」Tab 采用“下拉选择单个 Agent + 绑定按钮”的方式，批量绑定效率低。
- 需求改为在 Agent 列表上直接勾选绑定，并提供「全选」能力，减少重复操作成本。
- 本次优先复用现有 `/skills/assign` 接口，不引入后端新接口。

## 执行步骤

1. 梳理并调整绑定状态模型：以 `skillAgentsData` 为初始值，维护“当前勾选集合”与“初始集合”的差异。
2. 将绑定交互改为 Agent 列表勾选：每个 Agent 行显示 checkbox，支持单项切换并实时展示选中数量。
3. 新增「全选/取消全选」按钮：作用于当前可见 Agent 列表，保持交互确定性。
4. 提交时按差异批量调用绑定接口：新增项走默认绑定，移除项走 `enabled=false`，避免无效请求。
5. 回填页面反馈与文档：提交成功后刷新绑定数据；同步更新功能文档中的前端交互约定描述。

## 关键影响点

- 前端页面：`frontend/src/pages/Skills.tsx`（绑定 Tab 主体交互重构）。
- 前端服务：`frontend/src/services/skillService.ts`（预计无需接口变更，必要时仅补充类型）。
- 功能文档：`docs/feature/AGENT_SKILL.md`（更新绑定交互说明）。

## 风险与处理

- 风险：勾选集合与后端真实绑定状态出现短暂不一致（例如切换 Skill 后残留状态）。
  - 处理：以 `skill.id` 和 `skillAgentsData` 变化为重置触发条件，确保状态同步。
- 风险：全选语义歧义（当前页还是全量）。
  - 处理：明确为“当前可见列表全选”，并在 UI 文案中说明。
