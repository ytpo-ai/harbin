# 组织管理与公司治理模块下线计划

## 目标

删除当前“组织管理”和“公司治理”相关的前端与后端功能代码，保留公共基础能力，确保系统可构建、可运行，并为后续重构预留清晰边界。

## 执行步骤

1. 全量扫描前后端代码，定位“组织管理/公司治理”相关模块、路由、服务、菜单、权限与文档引用。
2. 确认删除边界：移除业务功能代码，保留可被其他模块复用的通用能力与基础设施。
3. 删除前端功能入口与页面实现，移除相关服务调用与类型定义引用。
4. 删除后端对应模块（控制器、服务、DTO、Schema/Model、模块注册）及 API 暴露入口。
5. 清理联动配置（路由、导航、权限点、接口文档、功能说明）并修复编译依赖。
6. 执行 lint/typecheck/build（及可用测试），修复删除后遗留错误，确保主流程可用。

## 关键影响点

- 前端：路由注册、导航菜单、页面组件、API 调用层、类型声明
- 后端：Nest 模块装配、控制器与服务、数据模型与 DTO、接口文档
- 权限与配置：权限 key、菜单可见性、常量枚举
- 文档：README、docs/features、docs/api、用户指南

## 风险与依赖

- 组织与治理能力可能被其他模块间接依赖（例如下拉数据源、统计聚合、权限判断），需要在删除时同步断开或替代。
- 若存在数据库层关联查询或索引依赖，删除接口后需要确认不会影响其他业务链路。
