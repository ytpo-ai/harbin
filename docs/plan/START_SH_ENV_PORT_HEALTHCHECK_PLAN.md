# start.sh 数据服务端口校验计划

## 1. 背景与目标

- 现状：根目录 `start.sh` 通过容器 name 判断 MongoDB/Redis 是否启动。
- 目标：改为基于 `backend` 环境变量动态解析端口并做端口监听校验，减少容器命名耦合。

## 2. 执行步骤

1. 在 `start.sh` 增加 `backend` 环境文件选择逻辑：优先 `.env.$ENV`，其次其他 `.env.*`（排除 `.env.example`），最后回退 `.env`。
2. 从选中的环境文件提取 `REDIS_PORT`，未配置或非法时回退 `6379`。
3. 从 `MONGODB_URI` 解析端口，解析失败时回退 `27017`。
4. 重构数据服务检查逻辑：使用端口监听作为就绪判定；容器名仅用于 `docker-compose up -d` 拉起。
5. 增强启动日志：输出所用 env 文件、解析端口与回退原因，便于排障。
6. 执行脚本语法检查（`bash -n start.sh`）并确认关键分支无语法错误。

## 3. 关键影响点

- 运维脚本：`start.sh`
- 配置解析：`backend/.env*`
- 启动可观测性：日志提示文案

## 4. 风险与依赖

- `mongodb+srv://` 或未显式端口的 URI 无法直接提取端口，将回退默认 `27017`。
- 若目标端口被其他进程占用，端口检测会判定为已就绪（需结合日志判断是否为目标服务）。
- 依赖本机可用 `lsof` 与 `docker-compose`。
