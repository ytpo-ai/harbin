# HR 账号全量展示与可编辑计划

## 背景

当前系统中，部分账号（如系统管理员）未在 HR 人类员工/人力资源页面中展示，导致无法在该页面统一维护账号信息。

## 目标

- HR 人力资源页面展示所有账号（包含系统管理员）
- 在 HR 页面可对这些账号进行编辑并保存
- 保持现有权限边界，避免影响高危字段与核心安全策略
- 在 HR 人类列表展示每位人类绑定的专属助理信息
- 移除 AI Agent 员工身份（后端能力移除并硬删除历史数据）

## 执行步骤

1. 梳理 HR 列表数据链路，确认后端与前端的筛选条件与角色过滤点
2. 定位并移除“系统管理员账号被排除”的逻辑（接口查询/服务层/前端二次过滤）
3. 调整接口返回与前端展示，确保所有账号进入 HR 人力资源列表
4. 调整编辑入口与保存逻辑，确保可编辑账号可在 HR 页面完成修改
5. 补充或更新测试，覆盖“全量可见 + 可编辑 + 保存成功”核心路径
6. 回归验证并检查文档（README/docs）是否需要同步更新

## 关键影响点

- 后端：账号查询条件、角色过滤、编辑权限校验
- 前端：HR 列表渲染、编辑按钮显示规则、编辑表单可用性
- 测试：接口与页面行为回归
- 文档：如接口行为或权限策略变化需同步说明

## 风险与依赖

- 若存在“超级管理员不可修改特定字段”的安全约束，需要保留并仅开放安全范围内字段
- 若查询逻辑被多个模块复用，需要评估改动对其他页面的影响

## 本轮补充范围（用户追加需求）

1. HR 人类员工列表新增“专属助理”展示（有绑定显示助理标识，无绑定显示未绑定）
2. 后端移除 AI Agent 员工创建与使用分支，员工域仅保留 human 类型
3. 启动后对历史 `type=agent` 员工记录执行硬删除，确保系统内不再存在 AI Agent 员工身份数据

## 本轮追加需求（专属助理名称联动）

1. HR 人类员工列表中“专属助理”由展示 Agent ID 改为展示助理名称
2. 员工接口返回增加专属助理名称字段（`exclusiveAssistantName`），由后端统一聚合
3. 当人类员工姓名变更时，若已绑定专属助理，后端同步更新助理名称
