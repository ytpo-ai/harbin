# Engineering Intelligence Project Statistics Plan

## 1. 背景与目标

- 在主前端 `研发智能` 模块下新增二级菜单 `工程统计`，提供一键统计能力。
- 统计覆盖 `docs`、`frontend`、`backend`，支持按项目明细展示与汇总展示。
- 统计结果落库到 MongoDB，支持历史追踪。
- 统计能力沉淀为可复用服务，并提供 Agent 可调用的 MCP 工具入口。

## 2. 执行步骤

1. 明确统计口径与统一返回结构（项目明细、汇总、错误信息、状态）。
2. 后端新增统计快照模型与索引，定义 `running/success/failed` 状态机。
3. 后端实现统计任务触发与计算链路，支持异步执行与状态查询。
4. 后端开放统计 API（创建、latest、详情、历史列表）。
5. 前端新增 `工程统计` 二级菜单和页面，接入按钮触发、汇总卡片、明细表、历史列表。
6. 在 Agents Tool 中新增 `engineering_statistics_run` MCP 工具，复用同一后端统计能力。
7. 更新功能/API/日常日志文档并补充必要验证。

## 3. 关键影响点

- 后端（EI）：统计领域服务、快照持久化、API 控制器。
- 后端（Agents）：新增内置 MCP 工具定义与执行分发逻辑。
- 前端：研发智能菜单结构、统计页面与服务调用。
- 数据库：新增统计快照集合与索引。
- 文档：feature/api/dailylog 需要同步更新。

## 4. 风险与依赖

- 大目录统计可能耗时较长，需要异步执行避免请求超时。
- Token 统计默认估算口径（chars/4），后续可扩展精算模式。
- 项目路径可能存在不可访问或失效场景，需要记录部分失败并继续汇总。

## 5. 完成标准

- `研发智能 -> 工程统计` 菜单可访问并可触发统计。
- 单次统计可落库并返回项目明细 + 汇总。
- 支持查询最新统计与历史统计。
- Agent 可通过 MCP 工具触发统计并获取结果。
- 相关文档按规范更新。
