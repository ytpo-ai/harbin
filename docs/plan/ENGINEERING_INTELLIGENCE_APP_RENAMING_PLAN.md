# 研发智能命名与边界调整计划

## 需求目标

将当前“CTO 文档感知”能力升级为独立的“研发智能（Engineering Intelligence）”应用，服务整个研发团队技术状态感知与评估，不再绑定单一角色。

## 执行步骤

1. 统一领域命名：将 `CTO` 相关命名替换为 `Engineering Intelligence`，包括后端模块、前端页面、服务命名与 API 路径。
2. 主应用入口：新增前端主应用路由 `/engineering-intelligence` 与菜单入口“研发智能”，明确与其他模块的边界。
3. 后端接口语义升级：将接口前缀调整为 `/engineering-intelligence`，并保留 `/cto-docs` 兼容入口避免已有调用中断。
4. 文案与输出语义调整：摘要输出从“CTO 感知”改为“研发技术状态感知”，避免角色绑定描述。
5. 文档同步：更新 README、API 文档与已有 Phase 1 计划文档术语，保持产品定位一致。

## 关键影响点

- 后端：模块/控制器/服务/Schema 命名与路由前缀。
- 前端：页面组件、接口服务、导航与路由。
- API：主路径变更并新增兼容路径说明。
- 文档：产品定位、接口说明、环境变量注释。

## 风险与依赖

- 路径改名会影响已接入调用方，需保留兼容路由。
- 当前仅处理“文档阅读与摘要”能力，后续评估与自动开发能力暂不实现。
