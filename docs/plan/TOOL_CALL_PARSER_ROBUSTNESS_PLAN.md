# 工具调用解析鲁棒性修复计划

> 状态：已并入统一主计划 `docs/plan/MEETING_CAPABILITY_MASTER_PLAN.md`（会议相关部分），本文件保留历史细节。

## 需求理解

- 当前会议场景中，`Model Management Agent` 可能输出不完整的工具调用标记（仅 `<tool_call>` 无 `</tool_call>`）。
- 现有解析器依赖完整闭合标签，导致工具未执行、原始控制指令泄露给用户。

## 执行步骤

1. 分析 Agent 工具调用解析逻辑，确认不完整标签与裸 JSON 的失败路径。
2. 增强工具调用提取逻辑：支持闭合标签、不闭合标签、裸 JSON 三种格式。
3. 增强工具标记清理逻辑：对不完整标签也能移除，避免返回控制指令文本。
4. 强化模型管理 Agent 提示词，要求工具调用严格闭合并单独输出。
5. 完成构建验证并回归会议场景。

## 关键影响点

- 后端：`agents` 服务中工具调用编排与解析方法。
- Agent行为：模型管理 Agent 的提示词规范。

## 风险与依赖

- 过度宽松解析可能误把普通 JSON 识别为工具调用，需要限定字段结构（`tool` + `parameters`）。
- 工具执行后仍需依赖模型二次回答，若模型继续输出格式错误，需清理兜底文本。

## 验证方式

- 会议中发送“搜索最新openai模型”，若出现未闭合标签也可成功执行工具。
- 最终用户可见消息不包含 `<tool_call>` 控制标记。
