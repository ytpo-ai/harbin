# Engineering Statistics History Snapshot Plan

## 1. 背景与目标

- 现有 `工程统计` 页面已具备触发统计与基础快照展示能力，但历史列表信息密度不足、缺少可操作入口。
- 本次目标是在不新增后端接口的前提下，补齐「历史统计列表 + 历史快照详情查看」完整链路。
- 前端交互遵循 `docs/FRONTEND_GUIDELINE.md`：列表优先、详情抽屉化、状态反馈一致、分页加载。

## 2. 页面设计方案（前端）

1. 页面采用「历史列表主视图 + 详情抽屉」结构，默认进入可直接浏览历史快照。
2. 顶部保留统计触发与刷新，增加 `状态筛选 + 条数筛选`，降低历史定位成本。
3. 历史列表使用表格承载核心字段（时间、状态、范围、模式、项目数、总字节、触发人、操作）。
4. 点击「查看快照」打开右侧抽屉，抽屉内使用 Tab：`汇总` / `项目明细` / `异常信息`。
5. 抽屉支持通过 `snapshotId` URL 参数直达，兼容消息中心跳转。

## 3. 执行步骤

1. 重构 `EngineeringStatistics` 页面信息架构，切换为历史列表优先视图。
2. 接入历史列表筛选与前端分页逻辑，统一空态/加载态/错误态与重试入口。
3. 增加快照详情抽屉组件区块与 Tab 切换，接入 `GET /statistics/snapshots/:snapshotId`。
4. 联动 `latest` 与 `history`：触发统计后自动刷新并支持定位最新快照。
5. 增加 `running` 状态自动刷新（轻量轮询）与 `failed` 错误信息聚合展示。
6. 校验移动端可用性（窄屏表格横向滚动 + 抽屉全宽）与交互闭环。

## 4. 关键影响点

- 前端页面：`frontend/src/pages/EngineeringStatistics.tsx`。
- 前端服务：复用 `frontend/src/services/engineeringIntelligenceService.ts` 现有快照接口。
- 路由与跳转：保留 `/engineering-intelligence/statistics?snapshotId=...` 深链能力。

## 5. 风险与应对

- 历史数据量增长：通过 `limit` 控制获取规模，并在前端分页展示。
- 快照字段历史兼容差异：详情区统一做空值兜底，避免渲染异常。
- 统计进行中频繁刷新：仅在存在 `running` 快照时启用轮询，控制请求频率。

## 6. 完成标准

- 页面默认展示可操作的历史统计列表。
- 列表支持按状态筛选、分页浏览与手动刷新。
- 每条历史记录可打开详情抽屉，查看汇总/项目明细/异常信息。
- 支持通过 URL `snapshotId` 打开对应快照详情。
- 统计触发后，列表与最新快照信息可自动同步更新。
