# Skill 管理前端优化计划

## 背景与目标

- 统一 Skill 管理页关键操作与信息密度，减少列表噪音并提升可编辑性。
- 前端 `skill` 字段与后端接口完全对齐，避免映射偏差和回显异常。
- 将 Agent 绑定纳入详情抽屉，形成「详情 / Agent 绑定」双 Tab 工作流。

## 执行步骤

1. 对齐前后端字段：梳理 `skill` 的类型、枚举、默认值与可选字段，统一前端模型和 API 映射，并将分类固定为「会议 / 计划 / 通用」。
2. 统一顶部交互：将「检索 skill」与「重建 skill 文档」收敛为「新增 Skill」右侧单图标菜单入口（下拉菜单承载操作项）。
3. 列表降噪：限制 `skill item` 高度（2-3 行 + 渐隐），并将标题/状态/操作统一放入 item header 栏，通过「查看详情」进入右侧抽屉查看完整内容。
4. 抽屉承载编辑：抽屉改为「详情 / Agent 绑定」Tab，支持直接编辑保存（乐观更新、失败回滚、保存中态）。
5. 操作语义合并：移除独立「取消状态」入口，将取消/恢复统一并入「修改」流程。
6. 中文化展示：状态与分类在列表、详情、筛选项统一中文展示，分类固定为「会议 / 计划 / 通用」。
7. 详情字段结构优化：详情 Tab 展示每个字段名（label），并将字段顺序调整为 `description -> metadata -> content`，其中 metadata 使用 Markdown 文档格式编辑。
8. 质量兜底：补充交互测试（操作菜单、抽屉保存、异常提示）与可访问性检查（焦点管理、ESC 关闭）。

## 关键影响点

- 前端页面：`frontend/src/pages/Skills.tsx`
- 前端服务：`frontend/src/services/skillService.ts`
- 前端类型：`frontend/src/types/index.ts`（或技能相关类型定义文件）
- 测试：Skills 页面相关交互与保存流程测试

## 风险与依赖

- 依赖后端最新 `skill` DTO/枚举（状态、分类、agent 绑定字段）的准确契约。
- 历史数据可能存在旧枚举值，需要前端映射兜底避免渲染异常。
- 乐观更新需保证列表与抽屉状态一致，避免并发编辑导致的 UI 回写冲突。
