# 旧版 Code MCP 工具下线计划

## 1. 需求背景

- 下线系统中的 4 个旧工具：
  - `code-docs-mcp`
  - `code-docs-reader`
  - `code-updates-mcp`
  - `code-updates-reader`
- 目标是让这些工具不再出现在工具列表、统一注册视图与可执行集合中。

## 2. 执行步骤

1. 定位这 4 个工具在 Tools 模块中的定义、注册、映射与白名单配置。
2. 删除工具定义与注册项，移除 legacy/unified 映射关系。
3. 清理 Agent 运行时与 MCP Profile 工具可见性配置中的相关引用。
4. 修复删除后的编译与类型引用问题，确保接口输出不再包含旧工具。
5. 补充/更新测试，覆盖工具列表与执行入口的回归场景。

## 3. 关键影响点

- 后端：`tools` 模块注册与执行编排逻辑
- API：`/tools`、`/tools/registry` 返回结果
- 治理：Agent/MCP Profile 白名单与映射
- 测试：工具查询与执行链路回归

## 4. 风险与依赖

- 若历史流程仍引用这些工具，下线后会出现“工具不存在/不可用”错误。
- 需同步清理遗留配置，避免运行时动态组装工具集时报错。

## 5. 验收标准

- 4 个目标工具在列表与注册接口中均不可见。
- 运行时不可执行这 4 个工具。
- 相关测试通过，且无新增类型/编译错误。
