# organizationId 下线后管理员会议准入与 HR 列表回归修复计划

## 背景

在后端移除 `organizationId` 后出现两处回归：

1. 管理员账号被误判为“未绑定专属助理”，导致无法发起或参与会议。
2. 前端「人力资源 > 员工列表」无内容展示。

## 目标

- 恢复管理员账号的正常会议准入（不因组织字段下线被误判）。
- 恢复 HR 员工列表展示（后端返回 + 前端渲染链路可用）。
- 仅做最小修复，不扩展权限边界与功能范围。

## 执行步骤

1. 排查会议准入判定链路，定位管理员误判触发点及 `organizationId` 遗留依赖。
2. 修复后端会议准入逻辑：移除无效组织上下文依赖，保留现有安全约束。
3. 排查 HR 员工列表数据链路：后端查询条件与前端二次过滤。
4. 修复后端员工列表查询条件，清理组织下线后的残留过滤导致的空结果。
5. 修复前端 HR 列表渲染/过滤逻辑，确保接口数据正常展示。
6. 执行回归验证（至少覆盖会议准入与 HR 列表展示），输出变更说明。

## 关键影响点

- 后端：会议准入策略、员工列表查询接口
- 前端：HR 员工列表渲染与空态判断
- 测试：会议参与、员工列表加载

## 风险与依赖

- 若网关/中间层仍有组织上下文强校验，可能需要补充同步清理。
- 修复需避免误放宽权限（仅消除误判，不改变权限模型）。
