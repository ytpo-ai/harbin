# 会议监控工具轮次环境化计划

## 背景

会议监控在同一轮巡检中需要调用多次 MCP 工具（`meeting.list`、`meeting.sendMessage`、`meeting.updateStatus`）。当前 Agent 工具轮次为硬编码常量，易在多会议场景触发“工具调用轮次已达上限”，导致只处理部分过期会议。

## 执行步骤

1. 将 `AgentService` 内 `maxToolRounds` 从硬编码改为读取环境变量 `MAX_TOOL_ROUNDS`。
2. 设置默认值为 `30`，并对非法值（非数字、非有限值、`<=0`）回退至默认值。
3. 在 `backend/.env.example` 增加 `MAX_TOOL_ROUNDS=30` 示例配置。
4. 更新会议功能文档配置项，补充该变量对会议监控批量处理能力的影响说明。
5. 运行一次类型/质量检查，确认改动可编译且无明显回归。

## 影响点

- 后端：`AgentService` 工具调用主循环
- 配置：环境变量与部署参数
- 文档：会议监控配置说明

## 风险与注意事项

- 轮次上限提高会增加单任务执行时长与模型/工具消耗。
- 若后续会议规模继续扩大，仍建议补充“批处理动作”与“幂等去重”能力，避免线性放大调用次数。
