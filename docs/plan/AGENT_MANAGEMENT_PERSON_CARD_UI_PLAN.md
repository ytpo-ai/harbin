# Agent 管理页人物卡片与创建顺序优化计划

## 需求目标

优化 Agent 管理相关页面交互：

1. Agent 列表改为人物卡片展示样式。
2. Agent 列表在无头像时显示默认头像。
3. Agent 创建页中“模型设置”区块移动到“工具设置”上方。

## 执行步骤

1. 梳理当前 Agent 列表与创建页组件结构，确认列表渲染单元、头像来源字段、创建表单区块顺序和提交映射。
2. 将 Agent 列表重构为人物卡片布局，保留现有核心信息与操作入口，并确保在桌面与移动端可读。
3. 为头像增加默认兜底逻辑：当头像为空或加载失败时展示默认人物头像。
4. 调整创建页表单区块顺序，将“模型设置”整体移动至“工具设置”前方，保持原有字段校验与提交结构不变。
5. 进行页面回归验证，重点检查列表展示、创建提交流程、区块切换与响应式效果。

## 关键影响点

- 前端：`Agents` 页面列表结构与样式、头像渲染逻辑、创建页表单区块排序。
- API：接口与数据结构原则上不变，仅调整前端展示层。
- 测试：需验证列表渲染、默认头像兜底、创建流程稳定性。
- 文档：如页面交互描述变化，需同步更新对应功能文档。

## 风险与依赖

- 卡片化改造可能影响现有操作按钮布局，需要保证操作可达性。
- 默认头像兜底需兼容不同数据格式（空字符串、null、无效 URL）。
- 创建页区块调整若与本地状态管理耦合，需避免字段值丢失或提交顺序异常。
