# Agent 编辑页面优化计划

## 需求目标

优化 Agent 编辑页面交互：

1. 将“模型设置”按钮和“编辑”按钮合并为统一入口。
2. 编辑区按 `模型` / `工具管理` / `基础信息` 三个 Tab 展示并可切换。
3. 支持编辑 Agent 的名称、Prompt、能力集等关键信息。

## 执行步骤

1. 梳理现有 Agent 编辑页的组件结构与交互流，定位“模型设置”和“编辑”按钮的触发逻辑，以及当前可编辑字段的数据来源。
2. 合并操作入口：将“模型设置”与“编辑”收敛为统一“编辑”入口，并移除或兼容旧入口路径。
3. 实现 Tab 化编辑容器：按 `模型` / `工具管理` / `基础信息` 组织表单模块，支持切换并保持未保存状态。
4. 完善可编辑项：在对应 Tab 中实现 `agent 名称`、`prompt`、`能力集` 等字段的编辑、校验与提交。
5. 联调与回归：验证保存成功/失败提示、切换 Tab 行为、脏数据保护；补充/更新相关测试。

## 关键影响点

- 前端：页面结构、按钮交互、表单状态管理。
- API：优先复用现有接口；如字段命名不一致，增加前端映射层。
- 后端/数据库：原则上不变（仅在能力集写入能力不足时评估改动）。
- 测试：更新 UI 交互与提交流程测试。
- 文档：如交互路径变化，更新使用说明文档。

## 风险与依赖

- 旧入口逻辑可能分散，需避免遗漏。
- 多 Tab 状态同步可能导致数据覆盖或丢失。
- 能力集字段在前后端命名不一致时，可能引发提交失败。
