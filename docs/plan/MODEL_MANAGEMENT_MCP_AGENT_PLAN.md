# 模型管理 MCP 工具与 Agent 建设计划

## 需求理解

- 为系统新增“模型管理 MCP 工具”。
- 创建“模型管理 Agent”，可从互联网搜索最新模型并添加到系统中。
- 需要保证模型数据标准化、可校验、可去重，并符合现有权限与模块边界。

## 执行步骤

1. 梳理现有模型管理、工具执行、Agent 编排与配置模块，确认可复用的数据结构与调用链。
2. 设计并实现模型管理 MCP 工具，提供“联网搜索最新模型”和“写入系统模型库”能力。
3. 新增模型管理 Agent，将“搜索 -> 过滤 -> 标准化 -> 入库”流程固化为可复用能力。
4. 增加去重与字段归一化策略（模型名、供应商、版本、发布日期、上下文长度等）。
5. 补充测试，覆盖正常流程与异常场景（外部搜索失败、字段缺失、重复模型、写入失败）。
6. 更新接口与使用文档，补充配置说明与调用示例。

## 关键影响点

- 后端：工具模块、模型模块、Agent 相关模块与服务编排。
- API：MCP 工具入参/出参与模型写入接口能力。
- 数据库：模型数据结构补充与唯一性约束（如名称+供应商+版本）。
- 配置：联网搜索源、超时与重试策略相关配置。
- 测试：服务层与工具执行链路测试。
- 文档：`README.md`、`docs/api/API.md` 与相关功能文档。

## 风险与依赖

- 外部数据源稳定性与结构不一致，需做超时、重试和解析容错。
- “最新模型”判断需定义规则（发布时间优先、官方来源优先）。
- 自动入库存在误写风险，需保证参数校验和权限控制。
- 不同厂商命名差异可能导致重复，需统一归一化规则。

## 验证方式

- 工具验证：可通过 MCP 调用搜索并返回结构化候选模型。
- Agent 验证：可一键完成“搜索并新增模型”闭环。
- 数据验证：重复模型不会重复写入，新增模型字段完整。
- 质量验证：通过 lint、typecheck、test（按项目现有脚本执行）。
