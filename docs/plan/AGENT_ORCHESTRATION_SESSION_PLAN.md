# Agent 任务编排与 Session 管理实施计划

## 需求目标

1. 系统具备任务规划能力：通过一句提示词，Planner Agent 可自动进行任务编排与拆解。
2. 拆解后的子任务可分配给其他 Agent 或员工（Human）。
3. 系统可统一管理所有 Agent 的 Session 生命周期与上下文。

## 执行步骤（按顺序）

1. 定义核心领域模型
   - 建立并统一 `Plan`、`Task`、`Assignment`、`Executor(Agent/Human)`、`Session`、`RunLog` 的数据结构与状态机。
2. 设计 Planner 工作流
   - 输入 `prompt` 后，生成任务清单与依赖关系（DAG），并输出可执行编排结果。
3. 实现编排引擎（Orchestrator）
   - 支持串行/并行执行、重试机制、超时控制、失败补偿和人工介入节点。
4. 实现任务分配机制
   - 基于能力标签、优先级、负载情况将任务分派给 Agent 或员工，并支持手动改派。
5. 实现 Session Manager
   - 统一管理所有 Agent Session：创建、恢复、续期、归档、上下文管理与会话隔离。
6. 提供管理接口
   - 提供 API（可选前端）用于查看 Plan、任务状态、Session 列表、执行日志，以及重试/终止等操作。
7. 测试与文档完善
   - 补充编排流程、并发、容错、Session 一致性测试，并同步更新 README 与 docs 文档。

## 关键影响点

- 后端/API（高影响）：新增 Planner、Orchestrator、Session Manager 相关模块与接口。
- 数据库（高影响）：新增计划、任务图、分配记录、会话与执行日志数据模型与索引。
- 前端（中影响，可选）：新增任务看板与 Session 管理视图。
- 测试（高影响）：新增端到端编排测试与异常恢复测试。
- 文档（中影响）：新增架构说明、接口说明与使用指南。

## 风险与依赖

- 任务拆解质量风险：提示词歧义可能导致拆解结果不稳定。
- 并发一致性风险：并行执行场景下任务状态更新与日志写入需要保证一致性。
- 长会话风险：Session 上下文增长导致成本和性能压力。
- 外部依赖：LLM 调用策略、队列/调度组件（如 BullMQ）、MongoDB 索引策略、权限体系。

## 推荐交付节奏（MVP）

第一阶段（MVP）优先交付：
- Planner 基础拆解
- Orchestrator 基础串并行执行
- Session Manager 基础能力（create/resume/list/archive）
- 后端 API 与最小测试闭环

后续迭代再补充：高级分配策略、人工审批流、可视化管理页面。

针对编排执行质量问题，追加以下能力：

1. 在 Orchestrator 执行器中读取依赖任务结果并组装执行上下文。
2. 将依赖摘要写入当前任务输入（description/messages/teamContext）。
3. 为外部动作任务增加“凭证输出协议”提示。
4. 执行后校验凭证，缺失时回退到人工处理状态。
5. 构建验证并记录接口行为变更。
