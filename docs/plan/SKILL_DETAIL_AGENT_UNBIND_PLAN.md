# Skill 详情页 Agent 解绑能力计划

## 需求目标

在前端 Skill 管理详情页的「Agent 绑定」视图中，为“已绑定 Agent”提供可执行的「解除绑定」能力，支持确认、提交、反馈与列表状态同步。

## 执行步骤

1. 梳理 `Skills` 页面中「Agent 绑定」Tab 的数据结构与请求流，确认已绑定列表渲染点与绑定接口复用方式。
2. 在已绑定 Agent 列表中新增「解除绑定」操作入口，并补充二次确认，避免误操作。
3. 复用技能绑定接口语义（`enabled=false`）实现解绑请求，接入 loading 状态与重复点击保护。
4. 完善操作反馈：解绑成功提示、失败提示与异常处理，确保用户可感知结果。
5. 在解绑成功后刷新当前 Skill 的绑定数据（或局部更新），保证 UI 与服务端状态一致。
6. 补充/更新前端测试，覆盖解绑入口展示、确认触发、成功刷新与失败提示路径。

## 关键影响点

- 前端页面：`frontend/src/pages/Skills.tsx`（Agent 绑定 Tab 交互与状态同步）。
- 前端服务：`frontend/src/services/skillService.ts`（解绑调用封装复用与类型对齐）。
- API：优先复用既有 `/skills/assign` 能力，不新增接口。
- 测试：Skills 页面交互与绑定管理相关测试用例。
- 文档：开发完成后按需更新功能文档与日常记录。

## 风险与依赖

- 若绑定列表缺少稳定主键（如 `agentId`），可能导致解绑目标不准确。
- 详情抽屉与列表若存在多源状态，可能出现解绑后局部视图未同步。
- 接口错误信息若不规范，前端提示需做兜底文案处理。
