# Prompt 管理：新增/筛选/复制删除与描述字段改造计划

## 背景

- 当前 Prompt 管理页缺少“新增系统 Prompt”能力，无法在页面直接创建新的模板组合（`scene + role`）。
- 当前筛选交互以手动输入为主，易出现输入偏差，且无法直观看到库内已有维度值。
- 当前缺少 Prompt 描述字段，无法在管理页明确每个 Prompt 的用途。
- 当前缺少复制与删除能力，不利于基于既有模板快速派生和清理。

## 目标

- 在 Prompt 管理页面增加“新增系统 Prompt（草稿）”能力。
- 将筛选项改造为“基于数据库现有数据的下拉选择”，降低误筛与空查概率。
- 新增 Prompt `description` 字段，用于描述模板作用。
- 新增 Prompt 复制与删除能力，提升模板运维效率。
- 保持现有发布、回滚、版本对比、审计能力不受影响。

## 实施步骤

1. 梳理并复用当前 Prompt Registry 前后端接口与数据模型，明确新增入口与最小字段校验。
2. 数据模型扩展：为 `prompt_templates` 增加 `description` 字段，并同步前后端类型定义。
3. 前端 Prompt 管理页新增“新建模板”交互（弹窗表单），支持填写 `scene`、`role`、`description`、`content`、`summary` 并调用草稿保存接口。
4. 后端新增（或扩展）筛选元数据接口，返回当前数据库可选的 `scene`、`role`、`status` 候选值（去重、排序）。
5. 前端筛选区改为 Select 下拉，候选项来自元数据接口，并与现有列表查询参数联动。
6. 前端新增“复制 Prompt”交互：从现有版本回填所有字段，允许全量修改后创建新草稿。
7. 后端新增“删除模板版本”接口，前端新增删除按钮与二次确认（默认仅允许删除 draft/archived）。
8. 补齐类型定义、异常提示、空值兜底（例如历史脏数据的空字符串过滤），确保页面在无候选数据时可回退查询。
9. 完成构建/测试验证，并更新相关文档（feature/api/dailylog）。

## UI 收敛补充

- 版本列表改为图标化操作栏（编辑/删除/复制/发布），减少列表区文案噪音。
- 编辑器与审计日志迁移到独立详情页，列表页仅承担筛选与快捷动作。
- 顶部文案调整：`新增系统 Prompt` 改为 `新增`；刷新按钮改为仅图标展示。

## 关键影响点

- 后端：`prompt-registry` 管理服务与控制器（筛选元数据、删除接口、description 字段入库）。
- 前端：Prompt 管理页筛选区、新建/复制弹窗、删除操作入口。
- API：Prompt Registry 文档需补充元数据、删除、description 字段说明。
- 测试：新增接口与页面行为的回归验证。

## 风险与依赖

- 历史数据中可能存在空 `scene/role`，需要在元数据接口中过滤，避免前端出现空选项。
- 删除已发布版本可能影响线上解析稳定性，默认限制为不可删除 published。
- 若候选项规模增长，需限制返回长度并保证索引利用，避免影响列表查询性能。
