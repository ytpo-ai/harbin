# Agent 角色分层协议梳理计划

## 需求理解

- 需要统一整理 Agent 与人类员工通用的角色分层协议。
- 角色分为三层：高管、执行层、临时工。
- 需要明确每层默认能力、任务分派边界、指挥权优先级与限制规则。
- 输出两份文档，分别落盘到 `docs/architecture/` 与 `docs/skill/`。

## 执行步骤

1. 将原始需求抽象为统一术语和规则项（层级、默认能力、禁止项、分派关系）。
2. 定义高管层规则：默认能力、可分派范围、协作边界与例外机制。
3. 定义执行层与临时工规则：可执行任务来源、工具权限基线、升级路径。
4. 补充“人类高管专属助理”的优先级与冲突仲裁机制，形成可落地条款。
5. 编写架构文档（设计与治理视角），落盘 `docs/architecture/AGENT_ROLE_TIER_ARCHITECTURE.md`。
6. 编写 Skill 文档（运行时行为视角），落盘 `docs/skill/AGENT_ROLE_TIER_SKILL.md`。
7. 自检两份文档术语一致性、边界完整性与可执行性。

## 关键影响点

- 角色治理：明确三层角色边界，降低任务分派和权限解释歧义。
- 协作流程：建立“高管 -> 执行层/临时工”的标准分派链路。
- 指挥权：明确人类高管专属助理在特定场景下的更高话语权。
- 运行约束：明确临时工无系统管理工具的默认限制。

## 风险与依赖

- “其他高管必须遵守专属助理指令”需要限定触发场景，否则可能导致跨域冲突。
- 当前为规范文档落盘，后续若接入系统权限与调度器，需要单独实现映射与校验。
