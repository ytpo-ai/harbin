# 创建会议弹窗移除“部门会议”类型 Plan

## 1. 需求目标

- 在前端创建会议弹窗中，移除“部门会议（department）”可选项。
- 保持后端能力与历史数据兼容，不做后端枚举删除。

## 2. 执行步骤

1. 定位会议页面中创建弹窗的会议类型选项渲染逻辑。
2. 将创建弹窗可选类型从通用类型列表中分离，过滤掉 `department`。
3. 保留会议列表/详情对 `department` 的展示映射，避免历史数据显示异常。
4. 执行前端构建校验，确认类型与编译通过。

## 3. 影响点

- 前端：`frontend/src/pages/Meetings.tsx`
- 文档：当前 plan 文档（本文件）

## 4. 风险与处理

- 风险：若直接删除全局类型映射，历史 `department` 会议可能显示为其他默认类型。
- 处理：仅在“创建弹窗可选项”层面移除，不删除全局展示映射。

## 5. 确认记录

- 用户已确认同意本 Plan，可执行开发。
